# QQ 机器人配置指南

Blockcell 支持通过 QQ 机器人与智能体进行交互。QQ 渠道支持两种连接模式：

- **WebSocket 模式（推荐）**：机器人主动连接 QQ Gateway，无需配置公网 Webhook 地址，非常适合本地开发、内网部署和调试。
- **Webhook 模式**：需要配置公网可访问的 URL，QQ 服务器会向该 URL 推送消息。

## 1. 创建 QQ 机器人

1. 访问 [QQ 开放平台](https://q.qq.com/) 并登录。
2. 在 **开发者后台** -> **机器人管理** 中，点击 **创建机器人**。
3. 填写机器人名称、简介等信息，选择机器人类型（沙箱/正式）。
4. 创建成功后，在 **基本信息** 页面，复制并保存 **AppID** 和 **AppSecret**。

## 2. 配置机器人 Intents

QQ 机器人需要订阅事件 intents 才能接收消息。Blockcell 默认订阅以下事件：

- **PUBLIC_GUILD_MESSAGES**：频道消息事件
- **GROUP_AND_C2C_EVENT**：群聊 @ 消息和私聊消息事件

> 注意：这些 intents 已在代码中内置，Blockcell 会自动向 QQ Gateway 声明订阅这些事件。

## 3. 配置 Blockcell

在 Blockcell 的配置文件（如 `~/.blockcell/config.json5` 或项目目录下的 `config.json5`）中，找到 `channels` 配置块，修改 `qq` 部分：

### WebSocket 模式（推荐）

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "你的_APP_ID",
      "appSecret": "你的_APP_SECRET",
      "environment": "production",
      "mode": "websocket",
      "allowFrom": []
    }
  }
}
```

### Webhook 模式

如果你有公网 IP，可以使用 Webhook 模式：

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "你的_APP_ID",
      "appSecret": "你的_APP_SECRET",
      "environment": "production",
      "mode": "webhook",
      "allowFrom": []
    }
  }
}
```

### 沙箱环境

在开发和测试阶段，建议使用沙箱环境：

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "你的_APP_ID",
      "appSecret": "你的_APP_SECRET",
      "environment": "sandbox",
      "mode": "websocket",
      "allowFrom": []
    }
  }
}
```

### 配置项说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `enabled` | boolean | 是 | 是否启用 QQ 渠道 |
| `appId` | string | 是 | 在 QQ 开放平台获取的 AppID |
| `appSecret` | string | 是 | 在 QQ 开放平台获取的 AppSecret |
| `environment` | string | 否 | API 环境：`production`（正式，默认）或 `sandbox`（沙箱） |
| `mode` | string | 否 | 连接模式：`websocket`（默认）或 `webhook` |
| `allowFrom` | string[] | 否 | 允许访问的用户 ID 白名单。留空 `[]` 表示允许所有用户 |

> 如果你通过 `blockcell gateway` 启用 QQ，还需要在 `config.json5` 中补一条 owner 绑定，例如：
>
> ```json
> { "channelOwners": { "qq": "default" } }
> ```
>
> 如果同一渠道配置了多个机器人账号，可以进一步使用 `channels.qq.accounts` 配合 `channelAccountOwners.qq.<accountId>`，例如：
>
> ```json
> {
>   "channelAccountOwners": {
>     "qq": {
>       "bot1": "default",
>       "bot2": "ops"
>     }
>   },
>   "channels": {
>     "qq": {
>       "enabled": true,
>       "defaultAccountId": "bot1",
>       "accounts": {
>         "bot1": {
>           "enabled": true,
>           "appId": "APP_ID_1",
>           "appSecret": "APP_SECRET_1",
>           "environment": "production",
>           "mode": "websocket",
>           "allowFrom": []
>         },
>         "bot2": {
>           "enabled": true,
>           "appId": "APP_ID_2",
>           "appSecret": "APP_SECRET_2",
>           "environment": "sandbox",
>           "mode": "websocket",
>           "allowFrom": []
>         }
>       }
>     }
>   }
> }
> ```
>
> 这样 `bot1` 会进入 `default` agent，`bot2` 会进入 `ops` agent。
>
> 也可以用 CLI 直接设置：`blockcell channels owner set --channel qq --account bot1 --agent default`。
>
> 否则 Gateway 会因为"enabled channel has no owner"而拒绝启动。

## 4. 交互方式

### 私聊消息（C2C）

用户可以直接搜索你的机器人并发送消息。机器人会通过 `C2C_MESSAGE_CREATE` 事件接收消息。

### 群聊 @ 消息

在群聊中，用户需要通过 `@机器人` 的方式触发机器人。机器人会通过 `GROUP_AT_MESSAGE_CREATE` 事件接收消息。

### 频道消息

在 QQ 频道中，机器人可以接收频道内的消息（需要订阅 `PUBLIC_GUILD_MESSAGES` intent）。

## 5. 连接模式对比

| 特性 | WebSocket 模式 | Webhook 模式 |
|------|---------------|-------------|
| 公网 IP | 不需要 | 需要 |
| 适用场景 | 本地开发、内网部署 | 生产环境、有公网服务器 |
| 连接方式 | 机器人主动连接 QQ Gateway | QQ 服务器推送消息到你的服务器 |
| 稳定性 | 依赖网络质量，支持自动重连 | 依赖公网服务器稳定性 |
| 配置复杂度 | 低 | 高（需要配置公网 URL、HTTPS 证书等） |

## 6. 常见问题

### WebSocket 连接失败

1. 检查 `appId` 和 `appSecret` 是否正确。
2. 检查网络是否能访问 `wss://api.sgroup.qq.com`（正式）或 `wss://sandbox.api.sgroup.qq.com`（沙箱）。
3. 查看日志中的错误信息，确认是否是认证失败。

### 消息发送失败

1. 确认机器人已通过 QQ 开放平台的审核。
2. 确认机器人已获得消息发送权限。
3. 沙箱环境只能向沙箱用户发送消息。

### Token 过期

Blockcell 会自动管理 Token 的获取和刷新，Token 有效期默认为 7200 秒（2 小时）。系统会在 Token 过期前 5 分钟自动刷新。

## 7. 注意事项

- QQ API 对消息发送有频率限制，Blockcell 内置了限流机制。
- 文本消息最大长度有限制，超长消息 Blockcell 会自动处理。
- 沙箱环境和正式环境的 API 地址不同，请确保 `environment` 配置正确。
- QQ 是腾讯的产品，API 服务器在中国境内，中国大陆用户可直接访问，无需配置代理。
