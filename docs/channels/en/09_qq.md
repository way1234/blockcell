# QQ Bot Configuration Guide

Blockcell supports interacting with agents through QQ bots. The QQ channel supports two connection modes:

- **WebSocket Mode (Recommended)**: The bot actively connects to the QQ Gateway, requiring no public webhook URL configuration. Ideal for local development, intranet deployment, and debugging.
- **Webhook Mode**: Requires a public URL configuration. QQ servers will push messages to this URL.

## 1. Create a QQ Bot

1. Visit the [QQ Open Platform](https://q.qq.com/) and log in.
2. Navigate to **Developer Console** -> **Bot Management**, click **Create Bot**.
3. Fill in the bot name, description, and other information. Select the bot type (sandbox/production).
4. After successful creation, copy and save the **AppID** and **AppSecret** from the **Basic Information** page.

## 2. Configure Bot Intents

QQ bots need to subscribe to event intents to receive messages. Blockcell subscribes to the following events by default:

- **PUBLIC_GUILD_MESSAGES**: Guild/channel message events
- **GROUP_AND_C2C_EVENT**: Group @ mentions and direct message events

> Note: These intents are hardcoded in the code. Blockcell automatically declares these intent subscriptions to the QQ Gateway.

## 3. Configure Blockcell

In Blockcell's configuration file (e.g., `~/.blockcell/config.json5` or `config.json5` in the project directory), find the `channels` configuration block and modify the `qq` section:

### WebSocket Mode (Recommended)

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "YOUR_APP_ID",
      "appSecret": "YOUR_APP_SECRET",
      "environment": "production",
      "mode": "websocket",
      "allowFrom": []
    }
  }
}
```

### Webhook Mode

If you have a public IP, you can use webhook mode:

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "YOUR_APP_ID",
      "appSecret": "YOUR_APP_SECRET",
      "environment": "production",
      "mode": "webhook",
      "allowFrom": []
    }
  }
}
```

### Sandbox Environment

For development and testing, it's recommended to use the sandbox environment:

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "YOUR_APP_ID",
      "appSecret": "YOUR_APP_SECRET",
      "environment": "sandbox",
      "mode": "websocket",
      "allowFrom": []
    }
  }
}
```

### Configuration Options

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `enabled` | boolean | Yes | Whether to enable the QQ channel |
| `appId` | string | Yes | AppID obtained from QQ Open Platform |
| `appSecret` | string | Yes | AppSecret obtained from QQ Open Platform |
| `environment` | string | No | API environment: `production` (default) or `sandbox` |
| `mode` | string | No | Connection mode: `websocket` (default) or `webhook` |
| `allowFrom` | string[] | No | Allowlist of user IDs. Empty `[]` allows all users |

> If you enable QQ through `blockcell gateway`, you also need an owner binding in `config.json5`, for example:
>
> ```json
> { "channelOwners": { "qq": "default" } }
> ```
>
> If the same channel carries multiple bot accounts, you can further use `channels.qq.accounts` together with `channelAccountOwners.qq.<accountId>`, for example:
>
> ```json
> {
>   "channelAccountOwners": {
>     "qq": {
>       "bot1": "default",
>       "bot2": "ops"
>     }
>   },
>   "channels": {
>     "qq": {
>       "enabled": true,
>       "defaultAccountId": "bot1",
>       "accounts": {
>         "bot1": {
>           "enabled": true,
>           "appId": "APP_ID_1",
>           "appSecret": "APP_SECRET_1",
>           "environment": "production",
>           "mode": "websocket",
>           "allowFrom": []
>         },
>         "bot2": {
>           "enabled": true,
>           "appId": "APP_ID_2",
>           "appSecret": "APP_SECRET_2",
>           "environment": "sandbox",
>           "mode": "websocket",
>           "allowFrom": []
>         }
>       }
>     }
>   }
> }
> ```
>
> This routes `bot1` to the `default` agent and `bot2` to the `ops` agent.
>
> You can also configure the bindings from CLI: `blockcell channels owner set --channel qq --account bot1 --agent default`.
>
> Otherwise Gateway refuses to start because the enabled external channel has no owner.

## 4. Interaction Methods

### Direct Messages (C2C)

Users can directly search for your bot and send messages. The bot receives messages through the `C2C_MESSAGE_CREATE` event.

### Group @ Mentions

In group chats, users need to `@bot` to trigger the bot. The bot receives messages through the `GROUP_AT_MESSAGE_CREATE` event.

### Guild/Channel Messages

In QQ guilds/channels, the bot can receive channel messages (requires `PUBLIC_GUILD_MESSAGES` intent subscription).

## 5. Connection Mode Comparison

| Feature | WebSocket Mode | Webhook Mode |
|---------|---------------|--------------|
| Public IP | Not required | Required |
| Use Case | Local development, intranet deployment | Production, public servers |
| Connection | Bot actively connects to QQ Gateway | QQ servers push messages to your server |
| Stability | Depends on network quality, supports auto-reconnect | Depends on public server stability |
| Configuration Complexity | Low | High (requires public URL, HTTPS certificate, etc.) |

## 6. Troubleshooting

### WebSocket Connection Failed

1. Check if `appId` and `appSecret` are correct.
2. Check if the network can access `wss://api.sgroup.qq.com` (production) or `wss://sandbox.api.sgroup.qq.com` (sandbox).
3. Check error messages in logs to confirm if authentication failed.

### Message Sending Failed

1. Confirm the bot has passed QQ Open Platform review.
2. Confirm the bot has message sending permissions.
3. Sandbox environment can only send messages to sandbox users.

### Token Expired

Blockcell automatically manages token acquisition and refresh. Token validity is 7200 seconds (2 hours) by default. The system automatically refreshes tokens 5 minutes before expiration.

## 7. Notes

- QQ API has rate limits for message sending. Blockcell has built-in rate limiting.
- Text messages have maximum length limits. Blockcell handles longer messages automatically.
- Sandbox and production environments use different API endpoints. Ensure `environment` is configured correctly.
- QQ is a Tencent product with API servers located in China. Users in mainland China can access the API directly without a proxy.
